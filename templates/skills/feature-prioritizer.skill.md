# FEATURE PRIORITIZER — P0/P1/P2/P3 Feature Classification

## PURPOSE

Classifies features by priority to prevent scope creep
and ensure MVP focus.

## TRIGGERS

- "Prioritize my features"
- "What should I build first?"
- "Classify features"

## CLASSIFICATION

### P0 — Must Have (App breaks without this)

- Authentication (login, signup, password reset)
- Core workflow (the one thing the app does)
- Data persistence (database)
- Error handling
- Basic security

### P1 — Should Have (Needed for MVP launch)

- User profiles
- Basic settings
- Email notifications
- Basic analytics
- Responsive design
- Accessibility basics

### P2 — Nice to Have (v1 if time permits)

- Advanced settings
- Dark mode
- Export/import
- Keyboard shortcuts
- Animations
- Advanced filters

### P3 — Future (Post-launch)

- AI features
- Integrations
- Mobile app (if web first)
- Advanced analytics
- Custom branding
- White-label

## WORKFLOW

1. List all desired features
2. Classify each as P0/P1/P2/P3
3. Validate with user
4. Build P0 first, then P1, then P2
5. Never build P2/P3 before P0/P1

## RULES

- ALWAYS classify before implementation
- ALWAYS validate with user
- NEVER implement P2/P3 before P0/P1
- NEVER add features without classification
